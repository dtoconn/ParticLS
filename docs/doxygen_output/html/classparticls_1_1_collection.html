Detailed Description

template<unsigned int dim>
class particls::Collection< dim >

Provides a place for storing and manipulating groups of general state objects and particles.

The two primary attributes of this class are the objects variable and the particles variable. The objects variables is a list of StateObjects that will be evolved in time. The particles variable is a list of particles that are used to compute pairwise particle-particle forces. Note that particles may or may not exist in both the objects and particles lists, but only particles in the objects list will be integrated in time. Thus,"fixed" particles can be achieve by adding them to the particles list, but no the objects list.

The other member variables and functions are primarily designed for manipulating the objects and particles lists.

@seealso Particle, StateObject, Searcher

◆ Collection()

template<unsigned int dim>
particls::Collection< dim >::Collection ( YAML::Node const &  options = YAML::Node())

Construct an empty Collection from a YAML node.

This constructor creates an empty (i.e., no objects) Collection using the information stored in the YAML::Node. Specifically, this constructor will look at the following sub-nodes:

  • options["Search"]
  • options["ContactLaw"]
  • options["BodyForces"]

The contents of the options["Search"] block will be used in Searcher::Create to construct a search algorithm. The contents of options["ContactLaw"] will be passed to ContactForce::Create to construct a default contact law. This will be used when new particle pairs are found and a new contact force is created between them. The options["BodyForces"] block is parsed to construct body force functions that are added to each particle when it is added to the collection. Each sub node in options["BodyForces"] is extracted and passed to BodyForce::Create to construct a body force for new particles. For example, options["BodyForces"]["Gravity"] could define a GravitationalForce and options["BodyForces"]["WindDrag"] could define a drag force function. Both of these force functions would be added to a particle when AddObject is called with a particle object.

Parameters
[in]optionsA YAML node containing information for constructing the searcher and particle forces.
