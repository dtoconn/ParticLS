\doxysubsection{Detailed Description}
\subsubsection*{template$<$unsigned int dim$>$\newline
class particls\+::\+Collection$<$ dim $>$}

Provides a place for storing and manipulating groups of general state objects and particles. 

The two primary attributes of this class are the {\ttfamily objects} variable and the {\ttfamily particles} variable. The {\ttfamily objects} variables is a list of State\+Objects that will be evolved in time. The {\ttfamily particles} variable is a list of particles that are used to compute pairwise particle-\/particle forces. Note that particles may or may not exist in both the {\ttfamily objects} and {\ttfamily particles} lists, but only particles in the {\ttfamily objects} list will be integrated in time. Thus,\char`\"{}fixed\char`\"{} particles can be achieve by adding them to the {\ttfamily particles} list, but no the {\ttfamily objects} list.

The other member variables and functions are primarily designed for manipulating the {\ttfamily objects} and {\ttfamily particles} lists.

Construct an empty \mbox{\hyperlink{classparticls_1_1_collection}{Collection}} from a Y\+A\+ML node. 

This constructor creates an empty (i.\+e., no objects) \mbox{\hyperlink{classparticls_1_1_collection}{Collection}} using the information stored in the Y\+A\+M\+L\+::\+Node. Specifically, this constructor will look at the following sub-\/nodes\+:
\begin{DoxyItemize}
\item {\ttfamily options\mbox{[}\char`\"{}\+Search\char`\"{}\mbox{]}}
\item {\ttfamily options\mbox{[}\char`\"{}\+Contact\+Law\char`\"{}\mbox{]}}
\item {\ttfamily options\mbox{[}\char`\"{}\+Body\+Forces\char`\"{}\mbox{]}}
\end{DoxyItemize}

The contents of the {\ttfamily options\mbox{[}\char`\"{}\+Search\char`\"{}\mbox{]}} block will be used in \mbox{\hyperlink{classparticls_1_1_searcher_a6c56abd1afa9d8b132cf0c22ded95323}{Searcher\+::\+Create}} to construct a search algorithm. The contents of {\ttfamily options\mbox{[}\char`\"{}\+Contact\+Law\char`\"{}\mbox{]}} will be passed to \mbox{\hyperlink{classparticls_1_1_contact_force_ad4b7a3005dd55c0faf7944562fe54272}{Contact\+Force\+::\+Create}} to construct a default contact law. This will be used when new particle pairs are found and a new contact force is created between them. The {\ttfamily options\mbox{[}\char`\"{}\+Body\+Forces\char`\"{}\mbox{]}} block is parsed to construct body force functions that are added to each particle when it is added to the collection. Each sub node in {\ttfamily options\mbox{[}\char`\"{}\+Body\+Forces\char`\"{}\mbox{]}} is extracted and passed to \mbox{\hyperlink{classparticls_1_1_body_force_a2176c7787d0dad7523f90a8feeca91fd}{Body\+Force\+::\+Create}} to construct a body force for new particles. For example, {\ttfamily options\mbox{[}\char`\"{}\+Body\+Forces\char`\"{}\mbox{]}\mbox{[}\char`\"{}\+Gravity\char`\"{}\mbox{]}} could define a \mbox{\hyperlink{classparticls_1_1_gravitational_force}{Gravitational\+Force}} and {\ttfamily options\mbox{[}\char`\"{}\+Body\+Forces\char`\"{}\mbox{]}\mbox{[}\char`\"{}\+Wind\+Drag\char`\"{}\mbox{]}} could define a drag force function. Both of these force functions would be added to a particle when {\ttfamily Add\+Object} is called with a particle object.
